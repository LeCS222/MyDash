# Тестирование фич MyDash

MyDash — статичное ванильное SPA без тест-фреймворка. Тестирование — это запуск
локального сервера и проверка приложения в браузере через MCP `cursor-ide-browser`.

## Чеклист

```
- [ ] 1. Запустить локальный сервер
- [ ] 2. Открыть приложение в браузере
- [ ] 3. Проверить консоль на ошибки
- [ ] 4. Проверить рендер всех виджетов
- [ ] 5. Протестировать новую фичу по сценарию
- [ ] 6. Проверить сохранение состояния (localStorage)
- [ ] 7. Проверить адаптивность и доступность
- [ ] 8. Составить отчёт PASS/FAIL
```

## Шаги

**1. Запуск сервера.** ES-модули и `fetch` требуют HTTP (не `file://`). Запусти
в фоне из корня проекта и не блокируй терминал:

```bash
npx --yes serve -p 3000
```

Если `serve` недоступен — `python -m http.server 3000`.

**2. Открыть приложение.** Через `browser_navigate` перейди на
`http://localhost:3000`. Сделай `browser_snapshot` для структуры страницы.

**3. Консоль.** Проверь ошибки JS/сети через `browser_cdp` (`Log.enable`,
`Runtime.evaluate`). Любая ошибка в консоли или проваленный `fetch` = FAIL.

**4. Рендер виджетов.** Убедись, что в `.widget-grid` появились карточки
`.widget-card` с заголовком и телом. Число карточек = длине `widgets` в конфиге.

**5. Сценарий фичи.** Проверь именно изменённое поведение. Примеры:
- часы — время идёт и совпадает с ожидаемым форматом
- погода — показаны температура (°C), описание (не «Неизвестно») и город
- заметки — ввод сохраняется и восстанавливается после перезагрузки

**6. Состояние.** Изменяй данные (заметка, тема), перезагружай страницу
(`browser_navigate` повторно) и проверяй, что значение восстановилось из
localStorage.

**7. Адаптивность и доступность.** Проверь ширины ~360px и ~1200px (нет
горизонтального скролла, сетка перестраивается), видимый фокус на интерактивных
элементах, наличие `aria-label` у иконочных кнопок. Ориентир — `.cursor/rules/ui.mdc`.

**8. Отчёт.** Заверши сервер. Верни строгий вердикт:

```
Вердикт: PASS | FAIL
Проверено: <что протестировано>
Проблемы: <список с шагами воспроизведения или «нет»>
```

## Правила

- Не редактируй исходный код — только тестируй и сообщай. Фиксы — задача основного агента.
- Не выноси вердикт PASS при ошибках в консоли или невоспроизводимой фиче.
- Всегда останавливай запущенный dev-сервер после проверки.
